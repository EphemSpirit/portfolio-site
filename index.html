Projects

Flight Booker


This project was completed as part of the curriculum for The Odin Project, and shows off Rails' capabilities with nested attributes! A person can select a flight from the dropdowns, and if any results are found in the database, they'll pop up on the right hand side; if not, a friendly message will apologize for not having any flights. Once a person clicks "Book this Flight", they'll be taken to a screen to fill out passenger information. One passenger field is dynamically generated for the number of passengers specified on the index screen! Nested attributes in action! It's a small airport, so there's only 10 flights at the moment. But every major airline was once a small regional outfit, you just gotta have faith. Smart money is on booking a flight from RLA to SFO on 5/18/21 - I heard there's a flight then, but mum's the word.

The Machine


For this project, the assignment was to build a lightweight Facebook clone! Full disclosure, I deleted my facebook account years ago so I was working off memory and probably styled more in line with Facebook's first release. But the important thing is that all the core functionality is there! User accounts (authenticated through Devise), posts, comments, likes, reciprocal freidnships, a dashboard, the whole kit and kaboodle. I also employed the Faker gem in order to give my user profiles a profile picture of Bill Murray because...well who doesn't love Bill Murray. The freidnships were extremely challenging to build but in the end I was able to build something I'm happy with and which lets users send friend requests, while letting the receiver of that request accept it or deny it.

Organizer


The motivation behind this project was my chronic habit of jotting things down on a piece of paper, and then forgetting where that piece of paper is when I need it; I also needed a project that would let me practice polymorphic model associations. This is a basic organization app where people can create an account and be taken to their dahsboard and have the option of creating a text/image/to-do/calendar post. I then added a tagging functionality, which is where polymorphism comes in! Users are then free to use the search functionality, which will return any post that has the given tag. I employed some more advanced namespacing of my controllers and routes to implement the polymorphism, so I'm pretty proud of how this one turned out! Now people can safely dump their photos/etc here until they find a more permanent home for them.

Deli Marketplace


Who doesn't love a good deli sandwich! Now people can peruse the menu of my favorite deli (not real), and find their lunch for the day! This website features a searchable menu with "fuzzy" matching - if you type in only part of a product name, that product will still show up in the search results. It also has a shopping cart functionality! There's no payment functionality, but I was proud of having built a working shopping cart which is based on session data. Each new session generates a new cart if one deosn't already exist! For the deli owners, I added an "admin" side of the website (located at /products), so the deli owners can see a list of all products, edit, add, delete, etc. those items and keep their menu up to date and current!
